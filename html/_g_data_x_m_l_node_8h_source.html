00016 // These node, element, and document classes implement a subset of the methods
00017 // provided by NSXML.  While NSXML behavior is mimicked as much as possible,
00018 // there are important differences.
00019 //
00020 // The biggest difference is that, since this is based on libxml2, there
00021 // is no retain model for the underlying node data.  Rather than copy every
00022 // node obtained from a parse tree (which would have a substantial memory
00023 // impact), we rely on weak references, and it is up to the code that
00024 // created a document to retain it for as long as any
00025 // references rely on nodes inside that document tree.
