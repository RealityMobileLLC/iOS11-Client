Detailed Description

Manages a video transmit session to a RealityVision Video Server.

The MotionJpegTransmitClient sends an unlimited length multipart Motion-JPEG stream inside the request entity of an HTTP PUT request.

Writing each frame is asynchronous. The delegate is notified when each write operation completes via the -writeDidComplete: message.

If the connection to the server is closed, the delegate is notified by the -clientClosedWithError: message.

To prevent buffer overflow and memory leaks, there can be only one outstanding write operation at a time. After each call to -writeJpegData: or -writeJpegData:withGpgga:, the caller must wait for the delegate to receive a corresponding -writeDidComplete: message before calling one of the -writeJpegData: methods again.
